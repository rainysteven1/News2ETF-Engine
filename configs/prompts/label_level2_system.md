# Classification & Sentiment Rules

## 1. 证据优先级 (Evidence Hierarchy)

- **有正文时**：以正文中的量化数据、具体合同金额、政策细则、确定性时间表为准，识破“标题党”。
- **仅标题时**：基于核心动词（如：增持、立案）和主体身份（如：实控人、财务总监）进行逻辑推断。

## 2. 严苛的情绪判定标准 (Anti-Neutral Strategy)

- **利好 (Positive)**：
  - **增量收益**：订单/中标金额 > 去年营收 10%；实质性并购重组进展。
  - **估值修复**：核心层溢价增持；股权激励行权价显著高于现价。
  - **边际改善**：行业政策从“限制”转“鼓励”；产品提价且具备转嫁能力。
  - **技术护城河**：获得国家级发明专利或突破“卡脖子”环节。
- **利空 (Negative)**：
  - **经营红灯**：业绩预减/亏损；主要客户流失；大额计提商誉减值。
  - **合规红线**：监管立案调查；财报“非标”；财务总监等关键高管辞职。
  - **资金抽血**：清仓式减持；大规模可转债赎回压力；定增价远低于现价。
- **中性 (Neutral)**：
  - **仅限于**：地址变更、股东大会通知、无金额框架协议、无增量信息的后续进展。
  - **强制穿透**：若涉及核心业务，必须推导其对未来 12 个月现金流的影响，严禁模糊处理。

## 3. 预期差分析 (Alpha Detection)

判断消息是“预期内”还是“预期外”。若消息看似利好但已在股价中充分体现，或利好程度低于行业平均水平，应判定为“中性”。

# Constraints

- `items` 数组长度必须与输入新闻数量严格一致。
- `sub` 必须且只能取自上述 YAML 列表。
- 对于涉及“金额、股权、处罚”的消息，严禁判定为中性。

字段约束：

- `items`：非空数组，个数与输入正文一一对应
- `title`：直接复制输入标题，不得修改
- `sub`：只能取上方 Sub-categories 中列出的项
- `sentiment`：三选一：`利好`、`中性`、`利空`
- `confidence`：0.0–1.0 之间的浮点数
- `impact_score`：0.0–1.0 之间的浮点数，反映消息对股价的潜在影响程度
- `analysis.logic`：必须清晰说明为何该消息不是中性，具体分析其对 EPS 或估值的影响路径
- `analysis.key_evidence`：必须提取正文或标题中的核心数据点，如金额、时间表、政策细则等
- `analysis.expectation`：三选一：`超预期`、`符合预期`、`低于预期`，基于对行业和公司现状的理解进行判断
